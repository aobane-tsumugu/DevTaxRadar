# DevTax Radar セキュリティ方針

DevTax Radarは、Claude Code／Codexの利用履歴を扱うローカルファーストの
Webアプリです。履歴にはプロンプト、ソースコード、ファイルパスなどの
機密情報が含まれ得るため、税務計算以前に「端末から出さない」ことを
主要なセキュリティ境界とします。

## 保護する情報

- Claude Code／Codexの元履歴
- プロンプト、応答、ソースコード
- 絶対パス、リポジトリ名、ブランチ名
- セッションID、メッセージID
- 実際のAIサブスク支払額
- 配賦結果、税務プロファイル、確認履歴

## MVPの信頼境界

```text
ユーザーのブラウザ
      │ 127.0.0.1 + CSRF token
      ▼
DevTax Radar local API
      ├─ Claude/Codex履歴をread onlyで走査
      └─ 正規化したメタデータだけをローカルSQLiteへ保存
```

通常利用ではクラウドAPI、telemetry、リモートDBへ接続しません。元履歴の
変更、プロンプト本文の抽出、公開デモへの実データ転用も行いません。

## 実装済みの防御

- サーバーは`127.0.0.1`にだけbindする
- 更新系APIはプロセス起動ごとに生成するCSRF tokenを要求する
- ブラウザが送る`Origin`は`127.0.0.1`または`localhost`だけを許可する
- APIリクエスト本文を64 KiBに制限する
- SQLiteはOS標準のアプリデータ領域へ保存する
- 履歴中の識別子は端末内のsaltでハッシュ化する
- プロジェクト表示名はローカルUI向けの明示的なopt-inでのみ取得し、
  絶対パスではなく最後のフォルダ名だけを保存する
- Provider料金、未取得利用率、プロジェクト分類はCSRF保護された設定APIから
  ローカルSQLiteへ保存する
- Git管理対象と公開fixtureを`privacy:check`で検査する
- CIとテストでは合成fixtureと一時ディレクトリだけを利用する

CSRF tokenは認証情報ではありません。同じユーザー権限で動作する悪意ある
ローカルプロセスからの防御は、OSのユーザー分離とファイル権限に依存します。

## 保存しない情報

正規化イベントへ保存できる項目はProvider、月、ハッシュ化識別子、モデル、
トークン内訳、スキーマ版、信頼度に限定します。次は保存しません。

- プロンプト／応答本文
- ソースコードやファイル内容
- 元のセッションID、メッセージID
- 元の絶対パス

## 開発・テスト時のルール

1. 実ユーザーの`~/.claude`、`~/.codex`をfixtureへコピーしない
2. fixtureは架空の値だけで手書きする
3. DBテストでは`DEVTAX_RADAR_DATA_DIR`を一時ディレクトリへ設定する
4. サーバーテストでは`HOME`と`USERPROFILE`も一時ディレクトリへ設定する
5. ローカルラベルのテストは合成履歴を一時HOMEへコピーして行う
6. Release前にsource archiveと配布物の双方へprivacy checkを実行する
7. スクリーンショットは合成データ、またはallowlist式エクスポートだけを使う

## 公開デモ

公開デモを作る場合は完全な合成データを既定とします。実データから共有用
エクスポートを作る機能は、次のallowlistだけを新しいJSONへ書き出す方式にし、
元オブジェクトから危険項目を削除する方式にはしません。

- Provider
- 月
- `Product A`等の匿名名
- 丸めた利用割合または利用指数
- デモ用月額
- 税務候補と判定理由

## 脆弱性の報告

公開リポジトリで実データの混入や外部送信を発見した場合、Issueへ機密情報を
貼らないでください。リポジトリ管理者が用意する非公開の報告経路を利用して
ください。正式公開時には`SECURITY.md`の連絡先とサポート対象バージョンを
更新します。

## 現在の制約

- ローカルマルウェアや同一ユーザー権限の別プロセスは脅威モデル外
- DB暗号化は未実装であり、端末・OSのディスク暗号化に依存
- Claude Code／Codexの履歴形式変更にはAdapter更新が必要
- privacy checkは多層防御の一部であり、公開前レビューを置き換えない
- 本プロダクトは税務判断や申告結果を保証しない
