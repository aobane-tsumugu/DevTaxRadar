# 共有ペルソナ適合監査

対象: [副業の税務処理相談](https://chatgpt.com/share/6a5b289e-7e24-83e8-871d-95bec85d58e2)

## 結論

DevTax Radarは、Claude Code／Codexの定額料金をProvider・月・プロダクト別に配賦し、当年費用・将来原価・要確認へ整理する基礎を解決しています。

一方、この相談者の最大の悩みである「旧版と新版の境界」「供用開始の証拠」「旧版残価の新版への引継ぎ」「AI以外の直接費を含む取得価額」は、ガイドを追加した段階であり、入力・永続化・計算の完全実装には未到達です。

## 今回反映した改善

- 「1セッションまで」「時間・トークン」という実装以上の表現を、Provider・月・プロジェクト集計／加重トークンへ修正
- 10万円以上20万円未満は、通常償却または3年一括の候補と明記
- AI以外の直接費が未入力であることを境界確認画面へ表示
- 取得価額、資本的支出、制作原価、前払費用を説明する税務QA画面を追加
- 旧版→新版の大幅仕様変更・旧版廃止・二重控除・残価0の確認をQAへ追加
- 公開前でも正式原稿や本番工程へ採用すれば供用済みとなり得ることをQAへ追加
- 未実装のCSVボタンを無効化し、開発中と明示

## 適合状況

| 課題 | 状況 | 次の改修候補 |
|---|---|---|
| Claude／Codex履歴と定額料金の配賦 | 対応済み | セッション単位の根拠永続化 |
| A／Bプロダクト、私用、未取得利用 | 一部対応 | 同一リポジトリ内の作業をセッション単位で再分類 |
| 新規取得価額／資本的支出／通常経費 | 一部対応 | 供用前後・直接対応・作業目的を実ダッシュボードの決定木へ完全接続 |
| 10万円／20万円境界 | 一部対応 | 外注費等の直接費入力を追加し、資産全体で判定 |
| 旧版→新版と旧版残価 | ガイド対応 | 移行日、大幅変更、旧版廃止、残価、二重処理防止を保存する移行台帳 |
| 供用開始・正式採用証拠 | ガイド対応 | テスト／正式採用、供用日、証拠メモの入力と保存 |
| 制作原価・前払費用 | エンジン／ガイドのみ | 実ダッシュボードへ金額算定と永続化を接続 |
| PC、家賃、電気、交通 | 未対応 | 固定資産台帳・共通費按分を別モジュールで追加 |
| 暗号資産益等の所得集計 | 未対応 | 所得区分と同年内集計を別機能として追加 |

## 優先順位

1. セッション単位の分類上書きと根拠保存
2. 供用開始・正式採用・旧版→新版移行台帳
3. AI以外の直接費入力と資産全体の境界判定
4. 税務候補エンジンと実ダッシュボードの完全接続
5. PC等の固定資産、共通費、所得集計

本監査は製品適合性の評価であり、個別の税務判断を確定するものではありません。
